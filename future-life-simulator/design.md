# Design Guidance

## App Visual Direction

The app should feel playful, bold, and slightly nostalgic, with a modern mobile product polish. The core visual metaphor is a **slot machine / randomizer experience**, so the interface should feel energetic, tactile, and rewarding without becoming visually noisy.

The design should balance:

- **Pixel-game charm** for the main theme and interactive moments
- **Soft, friendly readability** for normal UI content
- **iOS-style clarity** for layout, spacing, hierarchy, and usability

Avoid making the app look like a generic casino product. The mood should be fun, cozy, whimsical, and challenge-oriented rather than gambling-focused.

---

## Typography

Use two font systems:

### 1. Pixel Display Font

Use **Pixelify Sans** for pixel-style display text.

Use it for:

- App logo / wordmark
- Slot machine reel labels
- Mode names
- Big challenge result text
- Decorative headings
- Pixel-style badges or stickers
- Special UI moments such as “Spin”, “Challenge Unlocked”, “Go!”, “Do It”, etc.

Guidance:

- Use Pixelify Sans sparingly for visual impact.
- Do not use it for long paragraphs or dense UI text.
- Keep letter spacing slightly loose if needed for readability.
- Prefer larger sizes when using this font.

### 2. Normal UI Font

Use **Nunito Sans** for all standard interface text.

Use it for:

- Body text
- Buttons
- Navigation labels
- Descriptions
- Settings
- Cards
- Forms
- Empty states
- Tooltips
- Onboarding copy

Guidance:

- Nunito Sans should carry most of the product’s readable UI.
- Use rounded, friendly weights such as Regular, Medium, SemiBold, and Bold.
- Keep the tone soft and approachable.

---

## Type Hierarchy

Suggested hierarchy:

- **Hero / Result Text:** Pixelify Sans, 32–44px
- **Screen Title:** Pixelify Sans or Nunito Sans Bold, 24–32px
- **Section Title:** Nunito Sans Bold, 18–22px
- **Card Title:** Nunito Sans SemiBold, 16–18px
- **Body Text:** Nunito Sans Regular, 14–16px
- **Helper Text:** Nunito Sans Regular, 12–14px
- **Button Text:** Nunito Sans Bold or Pixelify Sans for special primary actions

For the main “Spin” button, Pixelify Sans can be used if the button is part of the slot machine interaction. For regular buttons, use Nunito Sans.

---

## Color Direction

Use a bright, playful palette with strong contrast and soft background colors.

Recommended feeling:

- Creamy or pastel background
- Bold accent colors for mode identity
- High-contrast primary action
- Fun secondary colors for stickers, icons, and slot-machine details

Avoid:

- Dark casino-style red/black/gold dominance
- Overly realistic metallic textures
- Serious finance/gambling visual cues
- Too many saturated colors competing at once

The app should feel more like a playful life challenge toy than a casino machine.

references: #FFF8E7 #FFCBA4 #C1E1C1 #B2D8F0 #C3A6DB #1f0432

## UI Style

The UI should combine:

- Rounded iOS-style cards
- Pixel-inspired decorative details
- Soft shadows
- Chunky buttons
- Clear spacing
- Light tactile depth

Use large rounded corners for cards and containers. Buttons should feel pressable and satisfying. Important interactive elements can have slightly exaggerated depth, outlines, or pixel-style borders.

Good visual elements to include:

- Pixel icons
- Sticker-like badges
- Slot-machine reels
- Dice / randomizer motifs
- Confetti moments
- Small animated-feeling accents
- Chunky dividers
- Soft gradient backgrounds

---

## Layout Principles

The app should be mobile-first and easy to use with one hand.

Prioritize:

- A clear central interaction
- Strong visual hierarchy
- One main action per screen
- Large touch targets
- Comfortable spacing
- Minimal cognitive load

The slot machine should be the hero component when present. Other UI elements should support it rather than compete with it.

Suggested structure for the main screen:

1. App header / mode selector
2. Slot machine visual area
3. Current mode or challenge category
4. Primary action button
5. Secondary supporting actions
6. Recent results, saved ideas, or small encouragement copy

---

## Slot Machine Component

The slot machine is the emotional center of the app.

It should feel:

- Playful
- Anticipatory
- Tactile
- Rewarding
- Slightly magical

Design guidance:

- Use reels with clearly separated columns or slots.
- Use Pixelify Sans for reel text when possible.
- Use icons alongside text to improve quick recognition.
- Add small decorative lights, outlines, or pixel corners.
- Keep the result area visually prominent.
- Avoid making it feel like real-money gambling.

The slot machine should communicate randomness, surprise, and playful decision-making.

---

## Mode Design

Each mode should have a distinct identity while still belonging to the same system.

Example modes:

- **Eat** — food exploration, restaurants, cooking ideas
- **Go** — places, activities, small adventures
- **Do** — tasks, habits, creative prompts, life challenges
- **Bold** — social courage, novelty, stepping outside comfort zone

Each mode can have:

- A unique icon
- A unique accent color
- A small label
- A reel vocabulary set
- A distinct result card style

Keep the mode system simple and scannable.

---

## Buttons & Interactions

Primary actions should feel satisfying and obvious.

The main randomizing action can be labeled:

- Spin
- Roll
- Pick for me
- Surprise me
- Let fate choose

For the slot-machine theme, **Spin** is the most direct and understandable. However, avoid overusing casino language elsewhere.

Button guidance:

- Use large rounded buttons.
- Make the primary button visually dominant.
- Add pressed states.
- Use subtle bounce or reel-spin animation if motion is available.
- Secondary actions should be quieter and less visually dominant.

---

## Icon Style

Icons should be simple, chunky, and playful.

Preferred style:

- Pixel-inspired or rounded sticker-like icons
- Clear silhouettes
- Minimal internal detail
- Strong contrast
- Consistent stroke or pixel grid logic

Avoid mixing too many icon styles. If using pixel icons, keep all mode icons in the same pixel style.

---

## Cards & Components

Cards should feel soft and friendly.

Use cards for:

- Challenge results
- Saved results
- Mode descriptions
- History
- Suggestions
- Empty states

Card style:

- Rounded corners
- Soft shadow or subtle border
- Light background
- Clear title and short description
- Optional icon or sticker detail

Result cards can be more expressive than regular cards.

---

## Copywriting Tone

The product voice should be playful, encouraging, and light.

Tone keywords:

- Fun
- Curious
- Warm
- Slightly cheeky
- Encouraging
- Not too serious

Good examples:

- “Let fate pick something fun.”
- “Tiny adventure unlocked.”
- “Spin for your next move.”
- “Feeling bold today?”
- “No overthinking. Just spin.”

Avoid:

- Language that sounds like gambling
- Overly motivational self-help copy
- Corporate productivity language
- Anything that makes the user feel pressured

---

## Accessibility

Even with a playful visual style, the app should remain accessible.

Requirements:

- Maintain strong text contrast.
- Do not rely on color alone to distinguish modes.
- Keep touch targets at least 44px high.
- Use readable font sizes.
- Avoid long text in Pixelify Sans.
- Make animations optional or gentle.
- Ensure all icons have clear labels where needed.

---

## Motion Guidance

Motion should make the app feel delightful, not overwhelming.

Use motion for:

- Slot reel spinning
- Result reveal
- Button press feedback
- Mode switching
- Confetti or celebration moments

Motion style:

- Snappy
- Bouncy
- Short
- Playful
- Not dizzying

Avoid long or excessive animations that slow down repeated use.

---

## Overall Design Goal

The final app should feel like a **playful life randomizer**: a small magical tool that helps users make decisions, try new things, and break out of overthinking.

It should be visually memorable because of the pixel slot-machine theme, but still polished, readable, and usable as a real mobile app.
